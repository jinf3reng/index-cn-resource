# index-cn-resource

## 项目简介

本项目是一个 HTML 页面归档仓库，用于存放和发布多个独立的、静态的 HTML 页面。这些页面不针对任何特定域名或网站，而是作为通用资源集合，供需要时参考或使用。

仓库中的每个 HTML 页面均为独立文件，不依赖外部资源（除标准网络字体或公共库外），可直接在浏览器中打开运行。

## 目录说明

```
.
├── README.md          # 本文件
├── pages/             # 归档的 HTML 页面目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/            # 公共资源（可选）
    └── style.css
```

- `pages/`：存放所有归档的 HTML 页面。每个页面应保持自包含，尽量减少外部依赖。
- `assets/`：存放多个页面共用的样式、脚本或图片等资源。如无共用资源，此目录可省略。

## 页面归档说明

- 所有 HTML 页面均为静态内容，无后端逻辑。
- 页面文件名应使用小写字母、数字和连字符，避免特殊字符。
- 每个页面建议在 `<head>` 中注明简单描述或版本信息，便于识别。
- 页面内容应遵守相关法律法规，不得包含违法、侵权或恶意信息。

## 维护说明

### 添加新页面

1. 将新的 HTML 文件放入 `pages/` 目录。
2. 确保文件命名清晰，与已有文件不冲突。
3. 如果页面需要共用资源，请将资源放入 `assets/` 并正确引用。

### 更新已有页面

- 直接修改对应 HTML 文件即可。
- 建议在文件内注明修改日期或版本号，以便追踪。

### 删除页面

- 如某页面不再需要，可直接从 `pages/` 中移除。
- 如有相关资源不再被引用，也请一并清理。
